# Quantum Lens Scenario

Radical multi-perspective deconstruction + solution engineering engine. Two engines, one scenario.

## Quick Overview

**Perception Engine** (Quantum Lens): Decomposes any input into atomic claims, runs them through parallel cognitive lenses with structurally enforced anti-convergence, detects interference patterns, identifies cruxes, and produces a Killer Question.

**Solution Engine**: Engineers concrete solutions from insights. Works backward from goals, eliminates barriers, produces executable adaptation roadmaps with confidence-banded scoring.

QL = observation (wave function, all possibilities). SE = collapse (measurement, forcing concrete reality). `/quantum-full` is not QL + SE. It's QL x SE.

**Key difference from other tools:**
- vs. brainstorming: Batch parallel (not conversational), all lenses at once, structured template output
- vs. DSV: GENERATES perspectives (not validates). Same quantum vocab, opposite purpose.
- vs. ideation tools: Deconstructs external content (not builds new ideas)

## Workflow

### Perception (Quantum Lens)
```
INPUT -> Phase 0 (intake + AoT decomposition) -> Phase 1 (parallel lenses) -> Phase 2 (interference detection) -> Phase 3 (convergence + persistence)
```

### Solution (Solution Engine)
```
INPUT -> S0 (intake + mode detect) -> S1 (system scan) -> [S1.5: deep dive if repo] -> S2 (engineering, hybrid delegation) -> [S2.5: cascade if contra+--cascade] -> S3 (DSV synthesis) -> S4 (persist + action)
```

### Combined (quantum-full)
```
INPUT -> QL (Phases 0-3) -> Smart Filter -> SE (S0-S4) -> Combined Report
```
